1 · Executive Summary

Silhouette is a personalized insight-retrieval engine for young professionals who feel stuck. A user describes their situation in a few sentences; Silhouette classifies the underlying "stuck state," retrieves one real, human-sourced insight from a curated library, explains in one line why it fits, and — when possible — links to the exact moment in a video or talk where that insight was spoken. The goal of each session is a single genuine aha moment, not engagement or time-in-app.

The product is built and functional: a Next.js + LangChain application with a six-state classifier, a scored retrieval engine with diversity controls, a presentation layer, a feedback loop, and a substantial back-office pipeline of roughly 45 scripts for discovering, evaluating, verifying, and promoting content. The corpus is being expanded to the threshold needed for real user testing.

Part 1 · What Silhouette Is

1.1 The product in one paragraph

Silhouette is context-aware insight retrieval. It is not a chatbot, a meditation or wellness app, a journaling tool, a content feed, or a therapy product. The user brings an emotional/situational moment; the system returns one curated, credible, human-sourced insight — a quote or passage from someone who navigated something similar — with a short, specific reason it applies. The conversation is the intake mechanism; the recommendation is the product; the curated library is the moat.

1.2 The user and the "aha moment"

The target user is a young professional (roughly 22–32) who is functional but stuck — low motivation, unclear direction, a post-achievement plateau — and who already consumes self-improvement content but can't find the right thing in the moment they need it. Success is defined narrowly and deliberately: the user enters in a stuck state, receives one insight plus a reason, and leaves with a real shift in perspective and a sense of a path forward. Usage is intentionally intermittent; the product optimizes for the quality of a single session, not retention.

1.3 The six stuck states

The corpus and classifier are organized around six soft, non-clinical "stuck states":

StateWhat it feels like
inaction-loopKnows what to do, can't get themselves to do it; avoidance and resistance.
engagement-droughtGoing through the motions; flat, disengaged, depleted (burnout-adjacent).
direction-collapseLost sense of direction or purpose; includes a "post-achievement" variant.
possibility-paralysisToo many options; can't commit; fear of the wrong choice.
identity-transitionIdentity built on a role or achievement that's gone or changing.
momentum-gapFell off, lost a streak, feels "behind," struggling to restart.

These are clusters for retrieval and framing, never diagnoses. Users can implicitly correct the classification.

1.4 The SIO and the resonance model

The unit of the library is a Structured Insight Object (SIO): a single insight extracted from a credible human source, stored as a markdown file with a rich YAML metadata header plus a short insight body. Beyond the stuck-state tag, every SIO carries a resonance profile on two dimensions that drive ranking:

  • insight_type — one of mechanism, story, reframe, permission.
  • voice_register — e.g. direct/challenging, expert/scientific, warm/affirming, vulnerable/personal, intellectual/measured.

This lets retrieval match not just topic but the kind of thing that will land for this person in this moment.

Part 2 · How the Project Is Structured

2.1 Tech stack

  • App: Next.js (App Router, React 19), TypeScript.
  • Agent: LangChain.js (createToolCallingAgent + AgentExecutor), OpenAI gpt-4o-mini.
  • Embeddings / retrieval: OpenAI text-embedding-3-small; in-memory vector store with custom scored search, resonance boosts, and MMR diversity.
  • External: Tavily (web-search fallback), YouTube Data API v3 + yt-dlp (media discovery & timestamp work).
  • Data: file-based corpus (markdown SIOs + JSON source records); feedback as JSONL.

2.3 Runtime architecture (a single chat turn)

The path a message takes, end to end:

  1. Client → API. ChatWindow POSTs the message + session to /api/chat.
  2. Feedback short-circuit. detectFeedbackIntent catches short "yes" / "show me something different" replies before the agent runs, so feedback never retrieves against itself. Negative feedback triggers a one-time retry that excludes already-seen SIOs and prefers a different insight_type.
  3. State classification. classifyState labels the message with a stuck state, confidence, and resonance hints; injected into the agent as a system note.
  4. Agent loop. The LangChain agent calls knowledge_base → scoredSearch (state filter + resonance boosts + MMR diversity, excluding seen SIOs) → picks the best insight_id → calls present_insight.
  5. Presentation. presentInsight validates attribution, generates the guarded "why this applies" (and bridge/credibility) lines, derives confidence/verification labels, and renders the insight markdown.
  6. Render. InsightCard parses the markdown into slots (excerpt, attribution, why, source link) and renders the verified media embed when present.
  7. Feedback. FeedbackRow posts a dwell signal to /api/feedback-signal, then the follow-up routes back through the short-circuit; all events land in feedbackLog.

2.4 The SIO data model

Each SIO's YAML header carries (abbreviated): identity (insight_id, source_id), source metadata (speaker, show/platform, title, date, URL), a full media block (provider, video_id, embed_url, timestamp_start_seconds, display_mode, media_verification_status, rights notes), classification (primary_state_tag, secondary_state_tags), resonance (insight_type, voice_register, credibility_tier, intensity_level), the content (key_claim, content_summary, attribution, the excerpt body), and review/honesty fields (transcript_verified, human_review_status, rights/usage notes). The same fields drive both retrieval ranking and honest presentation.

Part 4 · How to Make the Existing Tools Better

Each item is a concrete, evidence-based improvement drawn from the tools' own design docs and known limitations.

4.1 The SIO harvesting / discovery agent

  • The verbatim-verification bottleneck is the #1 scaling limit. The agent deliberately never sets transcript_verified: true — a human must confirm wording. Improvement: integrate official transcript sources (TED, YouTube captions via yt-dlp) into the pipeline so the agent can propose a verbatim match with evidence for one-click human confirmation, instead of fully manual checking.
  • TED transcripts are JavaScript-rendered and aren't retrievable by simple fetch. Improvement: a dedicated TED transcript fetcher (headless render or the TED transcript endpoint) so TED candidates can be auto-verified like other sources.
  • The Source Scout only emits queries; it doesn't search. Improvement: a supervised web/YouTube search step that actually returns and pre-ranks candidate sources for those queries (still human-gated).
  • The LLM resonance judge isn't calibrated. Improvement: calibrate the judge against human-rated examples once the corpus passes ~50 SIOs, and track judge-vs-human agreement.
  • Gap detection uses editorial thresholds, not real demand. Improvement: feed real user queries/feedback into gap detection so we harvest what users actually ask for.

4.2 The timestamp embedding agent

  • Coverage is thin and mostly manual. ASR auto-apply only fires on HIGH-confidence exact-quote matches; paraphrase SIOs need human listening. Improvement: combine ASR alignment with chapter markers and transcript search into a single confidence-scored moment finder, and expand auto-apply to well-aligned close paraphrases with a clear evidence trail.
  • No closed-loop validation after applying. Improvement: automatically run validate-media and a render check after each timestamp write, and flag any embed that doesn't actually start at the claimed moment.
  • Podcast uploads often lack chapters. Improvement: prioritize sources with transcripts/chapters during harvesting so the timestamp step has material to work with.

4.3 The retrieval engine & magnet controls

  • Magnets are mainly a "no-hint" phenomenon. Classifier intake hints already fix most cases; a few benign default-aligned dominators (e.g. Newport, Huberman) remain. Improvement: keep the promotion-time win-rate gate as the primary guard, and add exposure-aware scoring as the corpus grows so no single SIO is over-served.
  • Thresholds were calibrated on a tiny corpus (~19 SIOs). Improvement: recalibrate similarity thresholds, novelty thresholds, and resonance boosts against the expanded corpus and a real evaluation set.
  • MMR diversity is near-no-op at small sizes. Improvement: re-evaluate its effect once the corpus is ~111 SIOs.

4.4 The feedback loop

  • "Yes" is ambiguous (politeness vs. genuine aha); dwell time is the disambiguator. Improvement: keep dwell qualification, and add a light, optional qualitative trigger only if double-rejection stays high after enough sessions.
  • Feedback doesn't yet route into anything automatically. Improvement: wire analyze-feedback output into gap detection and a presentation-review queue (human-reviewed), closing the loop from signal to corpus/presentation change.

4.5 The presentation layer

  • Tier-1 structural wins are scoped but not yet shipped: a visible source-confidence chip, text-only/no-embed parity, a factual credibility line, and quieting the feedback so the insight is the last beat. Improvement: ship these; they reduce cognitive load and strengthen vicarious credibility at low risk.
  • The card re-parses markdown back into slots, which is brittle. Improvement: pass derived metadata (confidence, credibility) through the structured channel instead of the markdown string.

Part 6 · Risks, Open Questions & Readiness for User Testing

6.1 Key risks

  • Retrieval quality is the core unvalidated assumption. Everything depends on whether the system reliably surfaces a genuinely well-matched insight. This must be tested with real users in real stuck moments.
  • Copyright / verbatim integrity. Legacy SIOs include paraphrase reconstructions; the expansion enforces verified verbatim. The standing posture is metadata + short non-substitutive excerpts + link-out; full text only for licensed/in-house content.
  • Small-corpus evaluation. Thresholds and diagnostics were tuned on a tiny corpus; conclusions drawn before the expanded corpus is recalibrated may mislead.
  • Trust at first contact. The user pays a vulnerability cost before seeing value; the first insight has to land and feel human, not algorithmic.

6.2 What must be true before sending to users

  • QA gates (run June 4, 2026): validate-media — PASS, 0 hard violations, 0 warnings. test-magnet-risk — exits clean, no blocking magnet (one monitored default-path dominator, Shankar in identity-transition, diversified by intake hints in production). test-retrieval-calibration — 25/45 (56%), below the 70% target. The drop is mostly a stale-test artifact: ~20 new SIOs joined the library, and several now win queries whose hand-curated "acceptable answer" sets were written for the pre-expansion corpus (e.g. brooks-theyre-not-bored, shankar-identity-paralysis, benshahar-arrival-fallacy). A minority are genuine new over-winners (notably brooks-theyre-not-bored across several engagement-drought probes). Both point to the same fix: refresh the calibration acceptable-sets for the new corpus and retune the resonance/threshold weights. This recalibration is the one true gate remaining before beta.
  • Coverage (detect-gaps): all six states at target (111 total), every state×register cell populated, only one low-priority advisory (direction-collapse 20 vs momentum-gap 16 — a mild balance note, not a hole).
  • The manual smoke test passes end to end (preamble → insight → media → feedback → retry).
  • The closed-beta plan and first-three-tester plan are ready (both exist as guides).
  • Tier-1 presentation improvements are shipped (recommended, not blocking).
Bottom line. The architecture, tooling, process, and now the corpus (111 verified SIOs, media validated, coverage balanced) are all in place and mature for a v1. One gate remains before beta: a retrieval recalibration pass — refresh the calibration acceptable-sets for the expanded corpus and retune the resonance/threshold weights to lift the pass rate back toward the ≥70% target and tame the one or two new over-winners. After that, Silhouette is ready for a closed beta whose single job is to validate that the insight reliably lands.

Appendix

A · Glossary

  • SIO (Structured Insight Object) — the unit of the library: one curated insight + rich metadata.
  • Stuck state — one of six soft, non-clinical clusters describing the user's moment.
  • Resonance profile — insight_type + voice_register; the second retrieval dimension.
  • Magnet — an SIO that wins a disproportionate share of a state's queries; guarded by a promotion gate.
  • scoredSearch — the retrieval function: semantic similarity + metadata boosts + MMR diversity.
  • aha moment — the single success metric: a real shift in how the user sees their situation.
